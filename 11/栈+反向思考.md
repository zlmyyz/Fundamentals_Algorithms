栈+思维（正难反易）

### 题目描述

对于一个长度为 $k$ 的序列 $a_1, a_2, \dots, a_k$，若下标 $i$ 满足以下至少一个条件，则称 $a_i$ 是可见的：
- 从左往右看可以看到它，即对于所有 $1 \leqslant j < i$，都有 $a_j \leqslant a_i$；
- 从右往左看可以看到它，即对于所有 $i < j \leqslant k$，都有 $a_j \leqslant a_i$。

换句话说，如果 $a_i$ 左边没有比它大的数，或者右边没有比它大的数，那么 $a_i$ 就是可见的。

例如，在序列 $[1, 2, 5, 3, 4]$ 中，$1$ 是可见的，因为从左往右看可以看到它；而 $3$ 是不可见的，因为无论从左往右看还是从右往左看，它都会被更大的数挡住。

现在给定一个长度为 $n$ 的排列$^{[1]}$ $p$。对于每一个下标 $x$，请你求出有多少个子区间$^{[2]}$ $[l, r]$，满足 $l \leqslant x \leqslant r$，且将该子区间作为一个独立序列时，$p_x$ 是可见的。

---

### 【名词解释】

- **排列**$^{[1]}$：长度为 $n$ 的排列是由 $1 \sim n$ 这 $n$ 个整数、按任意顺序组成的数组，其中每个整数恰好出现一次。例如，$\{2, 3, 1, 5, 4\}$ 是一个长度为 $5$ 的排列，而 $\{1, 2, 2\}$ 和 $\{1, 3, 4\}$ 都不是排列，因为前者存在重复元素，后者包含了超出范围的数。
- **子区间**$^{[2]}$：从原序列中，连续地选择一段元素（可以全选，但不可以不选）得到的新序列。

---

### 输入描述

每个测试文件均包含多组测试数据。第一行输入一个整数 $T$ ($1 \leqslant T \leqslant 10^4$) 代表数据组数。每组测试数据描述如下：

第一行输入一个整数 $n$ ($1 \leqslant n \leqslant 5 \times 10^5$)，表示排列长度。

第二行输入 $n$ 个整数 $p_1, p_2, \dots, p_n$ ($1 \leqslant p_i \leqslant n$)，表示排列 $p$。

除此之外，保证单个测试文件的 $n$ 之和不超过 $5 \times 10^5$。

### 输出描述

对于每一组测试数据，新起一行输出 $n$ 个非负整数，第 $x$ 个整数表示下标为 $x$ 的元素是可见的子区间数量。

```
#include<bits/stdc++.h>
using namespace std;
#define ll long long
const int N=500005;
int a[N],L[N],R[N];
int main(){
    ios::sync_with_stdio(0);cin.tie(0);
    int T;
    cin>>T;
    while(T--){
        stack<int>st;
        int n;
        cin>>n;
        for(int i=1;i<=n;i++){
            cin>>a[i];
            L[i]=0;
            R[i]=n+1;
        }
        for(int i=1;i<=n;i++){
            while(st.size()&&a[st.top()]<a[i]){
                R[st.top()]=i;
                st.pop();
            }
            if(st.size())L[i]=st.top();
            st.push(i);
        }
        for(int i=1;i<=n;i++){
            ll total=1ll*i*(n-i+1);
            ll bad=1ll*L[i]*(n-R[i]+1);
            cout<<total-bad<<(i==n?"":" ");
        }
        cout<<"\n";
    }
    return 0;
}
```

这道题的核心算法思想是“正难则反”**加上**“单调栈”。直接去数哪些区间是“可见”的比较复杂，但反过来计算哪些区间是“不可见”的却非常容易。

我把你代码里的逻辑分步拆解一下：

### 1. 什么时候一个数会被彻底“挡住”（不可见）？

题目规定，只要左边没被挡住，**或者**右边没被挡住，它就是可见的。

反过来说，一个数 $a_i$ 在某个区间 $[l, r]$ 里是**不可见**的，必须同时满足两个条件：

- 该区间内，它左边有比它大的数。
- 该区间内，它右边也有比它大的数。

假设我们在整个数组中，找到 $a_i$ 左边**第一个比它大**的数的位置记作 $L[i]$，右边**第一个比它大**的数的位置记作 $R[i]$。

那么，只有当区间左端点 $l$ 把 $L[i]$ 包含了进去（即 $l \leqslant L[i]$），**且**右端点 $r$ 把 $R[i]$ 也包含了进去（即 $r \geqslant R[i]$）时，$a_i$ 才会被左右夹击，彻底不可见。

### 2. 数学公式推导（组合数学）

对于位置 $i$ 上的数：

- **包含它的总区间数量：**

  左端点 $l$ 可以在 $1 \sim i$ 之间任选（共 $i$ 种选择）。

  右端点 $r$ 可以在 $i \sim n$ 之间任选（共 $n - i + 1$ 种选择）。

  所以，包含它的总区间数为 `total = i * (n - i + 1)`。

- **让它不可见的“坏区间”数量：**

  左端点 $l$ 必须选在 $1 \sim L[i]$ 之间才能包住左边的大数（共 $L[i]$ 种选择）。

  右端点 $r$ 必须选在 $R[i] \sim n$ 之间才能包住右边的大数（共 $n - R[i] + 1$ 种选择）。

  所以，不可见的区间数为 `bad = L[i] * (n - R[i] + 1)`。

- **最终答案：** `可见区间数 = total - bad`。

### 3. 单调栈的妙用

代码中间那个 `for` 循环和 `while` 就是著名的**单调栈**模板，用来在 $O(N)$ 的时间复杂度内，一次性把所有元素的 $L[i]$ 和 $R[i]$ 全找出来。

C++

```
        int top=0;
        for(int i=1;i<=n;i++){
            // 如果栈顶的数比当前进来的数 a[i] 小
            // 说明当前数 a[i] 就是栈顶数右边第一个比它大的数
            while(top>0 && a[st[top]]<a[i]){
                R[st[top]] = i; 
                top--;
            }
            // 把比当前数小的都踢掉后，栈里剩下的第一个（也就是新栈顶），
            // 就是当前数左边第一个比它大的数
            if(top>0) L[i] = st[top]; 
            
            st[++top] = i; // 当前元素入栈
        }
```

- 如果一个数左边没有比它大的数，初始化的 `L[i] = 0` 刚好会让计算 `bad` 时的方案数变成 $0$（因为不可能被左边挡住）。
- 同理，右边没有比它大的数时，初始化的 `R[i] = n + 1` 也会让 `(n - R[i] + 1)` 变成 $0$。这完美契合了我们的公式。
